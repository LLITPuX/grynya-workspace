# Протокол: Аудит Документації

**Призначення:** Забезпечення актуальності, консистентності та структурної правильності всієї документації у Grynya Workspace.

**Коли застосовувати:**
- Перед завершенням великої сесії або етапу розробки (`session-close`).
- Після створення нових скілів, скриптів або протоколів.
- Якщо виникли сумніви щодо правильності структури воркспейсу.

## Кроки виконання

1. **Запуск автоматичного аудиту**
   Запустіть скрипт аудиту для перевірки всіх скілів:
   \`\`\`bash
   node .agents/skills/workspace-manager/scripts/audit-docs.js
   \`\`\`

2. **Виправлення автоматичних помилок**
   - **Червоні помилки (❌):** Їх потрібно виправити обов'язково (немає `SKILL.md`, невідомі протоколи тощо).
   - **Жовті попередження (⚠️):** Переконайтеся, що відсутність папок `scripts/` чи інших є свідомим рішенням; за потреби ініціалізуйте їх через `scaffold-skill.js` або просто додайте. Документуйте всі знайдені файли в `protocols/` у відповідних `SKILL.md`.

3. **Ручна перевірка кореневої документації**
   - Відкрийте `GEMINI.md`. Перевірте, чи не з'явилися в ході останніх сесій нові глобальні конвенції чи правила, які туди варто внести.
   - Відкрийте `README.md`. Переконайтеся, що опис архітектури, доступних баз даних (наприклад, FalkorDB) та сторонніх інструментів все ще актуальний.

4. **Завершення**
   - Після внесення всіх необхідних змін у файли та повторного успішного запуску скрипта аудиту (без ❌ та бажано без ⚠️), зафіксуйте свої дії.
   - Оновіть журнал сесії, вказавши, що аудит пройдено успішно.
