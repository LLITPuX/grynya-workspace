# Протокол: Створення скілів

## Призначення

Покроковий процес створення нових стандартних скілів для агента.

## Фаза 1: Збір вимог

Перевір, чи користувач надав достатньо деталей:
- **Мета скілу** — що саме скіл повинен робити?
- **Файли/структури** — з якими файлами чи проєктами скіл взаємодіє?
- **Назва** — запропонуй ім'я у форматі `kebab-case`

Якщо деталей не вистачає — **ЗУПИНИСЬ** та запитай.

## Фаза 2: Симуляція

Перед генерацією файлів:
1. Прослідкуй кроки виконання подумки
2. Визнач потенційні edge cases
3. Оновіть план на основі знахідок
4. Отримай підтвердження від користувача

## Фаза 3: Генерація

### Крок 1. Scaffold

Запусти скрипт для створення структури:
```bash
node .agents/skills/workspace-manager/scripts/scaffold-skill.js <назва-скілу>
```

Скрипт створить:
```
.agents/skills/<назва-скілу>/
├── SKILL.md           ← шаблон із placeholder-ами
├── protocols/         ← для окремих протоколів
├── scripts/           ← для допоміжних скриптів
├── examples/          ← для прикладів
├── resources/         ← для довідкових матеріалів
└── logs/              ← для логування досвіду
```

### Крок 2. Заповнення

Відкрий згенерований `SKILL.md` і заповни placeholder-и:
- `[Назва скілу]` → назва
- `[Короткий опис]` → description
- Додай протоколи у таблицю
- Створи файли протоколів у `protocols/`

### Крок 3. Валідація

Скрипт scaffold вже перевіряє структуру при створенні. Додатково можна запустити:
```bash
node .agents/skills/workspace-manager/scripts/scaffold-skill.js --validate <шлях>
```

## Правила для SKILL.md

- SKILL.md — це **індекс**, а не монолітна інструкція
- Протоколи виносити в окремі файли у `protocols/`
- Максимальна автоматизація: всі рутинні дії в протоколах повинні бути автоматизовані за допомогою JS скриптів у папці `scripts/`.
- Шаблон: `examples/skill-template.md`
- Помилки та рішення записувати в `logs/`
