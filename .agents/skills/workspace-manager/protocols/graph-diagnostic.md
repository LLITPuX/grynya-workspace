# Протокол: Діагностика графа FalkorDB

Цей протокол описує процедуру перевірки працездатності та цілісності графа Git-репозиторію в базі даних FalkorDB.

## Коли застосовувати
- Після розгортання або оновлення MCP-сервера FalkorDB.
- При підозрі на некоректне відображення структури репозиторію в LLM.
- При появі помилок під час інкрементальної синхронізації (`sync-git-update.js`).
- Планово перед завершенням важливих сесій розробки.

## Кроки виконання

### 1. Перевірка статусу контейнера
Переконайтеся, що контейнер `falkordb` запущений та доступний:
```powershell
docker ps | findstr falkordb
```

### 2. Запуск скрипта діагностики
Виконайте скрипт з кореневої папки воркспейсу:
```powershell
node .agents/skills/workspace-manager/scripts/graph-diagnostic.js
```

### 3. Аналіз результатів

| Показник | Очікуваний результат | Дія при відхиленні |
|----------|----------------------|--------------------|
| **Docker/Ping** | ✅ Спрацювало | Перезапустіть Docker: `docker compose up -d` |
| **Вузли/Зв'язки** | Кількість відповідає реальному репозиторію | Виконайте повну синхронізацію |
| **Сирітські файли** | 0 знайдено | Перевірте шляхи в `sync-git-init.js` |
| **Сирітські комміти** | 0 знайдено | Можливо, комміт належить файлу, що видалений |

## Вирішення проблем

### Повна переініціалізація графа
Якщо діагностика виявила критичні розриви (багато Files/Folders без батьків), виконайте:
```powershell
node .agents/skills/workspace-manager/scripts/sync-git-init.js
```
Це очистить базу та заново просканує репозиторій.

### Оновлення MCP-конфігурую
Якщо скрипт не може під'єднатися до FalkorDB, перевірте, чи відповідає `mcp_config.json` поточному порту контейнера. Скрипт синхронізації конфігу:
```powershell
node .agents/skills/workspace-manager/scripts/update-mcp-config.js
```

## Логування
Результати діагностики слід фіксувати в `journal.json` поточної сесії у разі виявлення критичних помилок.
