# Протокол: Обслуговування інфраструктури

## Призначення
Підтримка стабільності та чистоти робочого середовища Grynya.

## Регулярні перевірки (Healthcheck)

Виконуються на початку кожної сесії (вбудовано у `session-open`):
1. **Docker Status**: `docker compose ps` — всі сервіси мають бути `Up`.
2. **Logs Checking**: `docker compose logs --tail=20` — перевірка на критичні помилки.
3. **MCP Sync**: `node ...update-mcp-config.js` — актуалізація конфігу.

## Очищення (Cleanup)

Раз на тиждень або при виникненні проблем із місцем:
1. **Тимчасові файли**: Очистити `/tmp/` та `.gemini/antigravity/tmp/`.
2. **Docker Prune**: `docker system prune -f` (тільки якщо впевнені, що не видалите потрібні томи).
3. **Artifact Management**: Архівація старих `walkthrough.md` та `logs/journal.json`, якщо вони стають завеликими.

## Робота з мережею
Всі контейнери повинні знаходитися в одній мережі `grynya-network`. Якщо контейнер не бачить інший:
- Перевірте секцію `networks` у `docker-compose.yml`.
- Використовуйте назви сервісів як хости (наприклад, `http://firecrawl-api-1:3002`).

> [!CAUTION]
> Ніколи не комітьте файл `.env` із секретами (API ключами) у публічні репозиторії. Додайте його в `.gitignore`.
