# Протокол: Відкриття сесії (Session Open)

## Призначення
Отримання контексту з попередніх сесій для продовження роботи без втрати досвіду. Виконується **на початку** кожної робочої сесії для кожного скілу, який планується використовувати.

## Алгоритм

1. **Запустити скрипт:**
   ```bash
   node .agents/skills/workspace-manager/scripts/session-log.js open <назва-скілу>
   ```

2. **Проаналізувати:**
   - Що було зроблено минулого разу?
   - Які були помилки? (секція `lessons` та `errors`)
   - Які були заплановані наступні кроки? (`next_steps`)

3. **Сформувати робочий план:**
   На основі отриманого контексту підкоригувати поточний `task.md` або `implementation_plan.md`.

> [!TIP]
> Якщо `journal.json` порожній, це означає, що ви першим створюєте історію цього скілу. Почніть з чистого аркуша, але будьте уважні до деталей для наступних сесій.
