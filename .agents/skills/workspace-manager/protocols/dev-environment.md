# Протокол: Середовище розробки (Hot Reload)

## Призначення
Забезпечення миттєвої синхронізації змін між Windows та Docker-контейнерами без необхідності перезбірки образів.

## Механізм Bind Mounts
У `docker-compose.yml` корінь воркспейсу монтується так:
```yaml
volumes:
  - .:/app
```

## Налаштування Hot Reload (Windows)
Через особливості Windows-файлової системи, `inotify` у Linux-контейнерах часто не спрацьовує на змонтовані томи.

**Рішення (Polling):**
Усі Node.js/Chokidar базувані вочери повинні використовувати режим поллінгу. Для цього у `environment` додаються:
- `CHOKIDAR_USEPOLLING=true`
- `WATCHPACK_POLLING=true`

## Вирішення проблем (Troubleshooting)

1. **Зміни не застосовуються**:
   - Перевірте, чи запущений Docker Desktop.
   - Запустіть `docker compose restart grynya-workspace`.
   - Переконайтеся, що файл не заблокований іншим процесом на Windows.

2. **Помилка MODULE_NOT_FOUND**:
   - Це стається, якщо ви монтуєте папку (`/app`), яка перекриває скомпільований код (`dist/`) всередині образу.
   - **Рішення**: Монтуйте конкретні папки (`/app/src`, `/app/docs`) або використовуйте `.dockerignore`.

> [!TIP]
> Якщо ви розробляєте скрипти у `.agents/skills/`, вони підхоплюються миттєво, оскільки зазвичай запускаються через `node` напряму.
