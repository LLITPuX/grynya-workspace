# Протокол: Закриття сесії (Session Close)

## Призначення
Фіксація досягнутого прогресу, отриманого досвіду та виправлених помилок. Виконується **перед завершенням** кожної робочої сесії.

## Алгоритм

1. **Запустити скрипт фіналізації:**
   ```bash
   node .agents/skills/workspace-manager/scripts/session-log.js close <назва-скілу> "<короткий-звіт>"
   ```
   > [!NOTE]
   > Ця команда виконує повний цикл:
   > 1. Збирає список змінених файлів.
   > 2. Заповнює JSON-файл сесії у `logs/`.
   > 3. Створює Git-коміт із посиланням на цей файл.
   > 4. Оновлює FalkorDB (через `sync-git-update.js`).
   > 5. Запускає `graph-diagnostic.js` для перевірки цілісності.

2. **Перевірка результатів діагностики:**
   Якщо діагностика виявила помилки (наприклад, порожні описи файлів), виправте JSON-файл вручну та запустіть `node .agents/skills/workspace-manager/scripts/session-log.js close` ще раз (або виконайте `sync-git-update.js` вручну).

3. **Аудит документації (Documentation Audit):**
   Перед завершенням перевірте:
   - чи потребує оновлення `README.md` або `SKILL.md`?
   - чи додані нові протоколи до таблиць?
   - чи актуальні коментарі до нових/змінених скриптів?

> [!IMPORTANT]
> Тільки після успішної діагностики сесія вважається повністю зафіксованою в базі знань проекту.
