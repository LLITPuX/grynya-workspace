# Протокол: Закриття сесії (Session Close)

## Призначення
Фіксація досягнутого прогресу, отриманого досвіду та виправлених помилок. Виконується **перед завершенням** кожної робочої сесії для кожного скілу, що був використаний.

## Алгоритм

1. **Запустити скрипт фіналізації:**
   ```bash
   node .agents/skills/workspace-manager/scripts/session-log.js close <назва-скілу> "<короткий-звіт>"
   ```
   > [!NOTE]
   > Це автоматично оновить статус вашої поточної сесії на **"Завершено"** та додасть ваш звіт.

2. **Заповнити технічні деталі (Lessons/Errors):**
   Відкрийте `.agents/skills/<назва-скілу>/logs/journal.json` та заповніть масиви `errors` та `lessons` для зафіксованого запису. Це робиться **строго один раз** перед завершенням.

3. **Аудит документації (Documentation Audit):**
   Перед фінальним комітом перевірте:
   - чи потребує оновлення `README.md` або `SKILL.md`?
   - чи додані нові протоколи до таблиць?
   - чи актуальні коментарі до нових/змінених скриптів?

4. **Зберегти та закомітити:**
   Журнал та оновлена документація мають бути частиною Git-репозиторію.

> [!IMPORTANT]
> Будьте чесними в описі помилок. Це ваш "бортовий журнал", який допоможе вам (або іншому агенту) уникнути тих самих граблів у майбутньому.
